无法通过 file:// 直接打开

浏览器安全策略禁止在本地文件协议下加载应用脚本（CORS / ES Module 限制）。这是正常现象，不是项目损坏。

本地预览：回到项目根目录，双击运行

serve-site.bat

然后在浏览器访问 http://127.0.0.1:8080

  1. 先运行 build-static.bat 生成 site 目录
  2. 再运行 serve-site.bat 启动本地服务
  3. 将 site 目录上传到 Web 服务器后，通过 http/https 访问即可
部署到 Nginx / OSS / GitHub Pages 等静态托管后，直接访问域名即可，无需此步骤。
